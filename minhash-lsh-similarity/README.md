# MinHash and Locality Sensitive Hashing Similarity Search

This module implements MinHash and Locality Sensitive Hashing (LSH) with PySpark to identify similar users based on the businesses they reviewed.

## Goals

- Convert review data into binary user-business interaction sets.
- Generate MinHash signatures for users.
- Use LSH to create candidate user pairs efficiently.
- Compute exact Jaccard similarity only for candidate pairs.
- Balance precision, recall, and runtime through LSH banding parameters.

## Task Overview

The project identifies similar user pairs from Yelp-style review data. A user is represented as a set of businesses they reviewed. Similarity between users is measured using Jaccard similarity:

```text
Jaccard Similarity = |intersection(user1, user2)| / |union(user1, user2)|
```

Because comparing every possible user pair is expensive, the project first uses MinHash and LSH to generate a smaller candidate set. Exact Jaccard similarity is then computed only for those candidates.

## Implementation Steps

### 1. Build Binary Rating Matrix

Each user is represented by a binary vector over businesses:

- `1` if the user reviewed the business
- `0` otherwise

### 2. Generate MinHash Signatures

Each user's business set is compressed into a fixed-length MinHash signature using 100 hash functions. The implementation uses a double-hashing scheme to generate multiple hash functions efficiently.

### 3. Apply LSH

The signature matrix is divided into bands and rows. If two users have the same values in at least one band, they become a candidate pair.

### 4. Compute Exact Jaccard Similarity

For each candidate pair, exact Jaccard similarity is computed using the original business sets. Only pairs above the threshold are written to the final output.

## Example Command

```bash
python task.py --input_file tr1.json --candidate_file candidate.out --output_file task.out --time_file task.time --threshold 0.3 --seed 0
```

## Output

The program writes two output files:

1. **Candidate file**: user pairs generated by LSH.
2. **Output file**: user pairs whose exact Jaccard similarity is above the threshold.

Each output line is a JSON record containing user IDs and, for final results, the similarity score.
